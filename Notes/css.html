In order to space things out from other tags and space text from borders, we alter a few things:

A margin is the space around an tags (box) that pushes it away from other boxes / edges.

A padding is the space between the border of a box and the text / tags inside. For example, the spacing between this text and this box.

A border also adds on additional spacing.

Units

Ems (em) It is a scalable unit. It gets bigger as you zoom in on webpages. Hence, it does not mess up the layout of the webpage. (Preferable to always use this)
Pixels (px) These are based on the pixels on the screen. Hence it does scale / resize when zooming in / out. As a result, zooming in and out makes no difference
Points(pt) Points are traditionally used in print media (anything that is to be printed on paper, etc.). One point is equal to 1/72 of an inch. Points are much like pixels, in that they are fixed-size units and cannot scale in size.
Percent (%) Percentages are exactly like em. They are scalable. However, the only difference is when the text setting is set as smallest and largest, em becomes way smaller and way bigger too abruptly as compared to percetages.

Floating and Display

To make things align side by side each other, we use float. To use this, the main div should contain 2 smaller divs. In CSS, one small div should float left and the other float right. It is better to give the main div a definite width so its cleaner.

Alternatively, to combine different words with different effects on the same sentence, we can use something called span. Using a bunch of "span" will combine the words on the same line. Even images can be inserted and it'll apear next to a span. The reasoning is given below:

The display of a tag determines the amount of space it occupies. Block displays like "p" or "div" are block and hence occupy the entire width of the screen, preventing any other inserted tags to appear next to it. Inline display like "span" causes it to occupy only the width of the tag hence allowing any other inline tags to appear next to it. "inline-block" displays are similar to that of inline except it allows the tag to have a modified width if modified in the CSS.

Positioning

There are different types of poisitioning and its effect. It is very useful for moving tags / divs / boxes / images to be anywhere on the screen.

STATIC

By default a tag's position is static.

Relative Positioning

It moves the tag RELATIVE to what it would've been. It goes with the flow of the body. - This means that once moved, the other tags stay where they are. However, a "ghost" of it occupies its previous space and hence any text put there will wrap around it.

Absolute Positioning

Absolute positioning causes it to move from the parent (the entire body). Hence, using "right" causes it to move FROM the RIGHT of the screen. It does not go with the flow of the webpage. Hence, writing "absolute" causes the tags around it to behave like that tag was not even there.

Fixed Positioning

Fixed position causes the tag / box etc... to remain essentially stuck to the user's webpage. Hence, scrolling through the webpage will cause the webpage to flow behind that fixed position tag.

Controlling Position

To control the position of tags on the screen, put it to any of the position types aboce (preferably absolute) then use either top, left, right to control where the position of the tag is. A value of top: 5em; will cause the image to move from the top by 5em. Similarly, a positive left or right value moves the tag right or left respectively.

Children & Parent tags

Children tags sit on top the parent tags. Eg: a span will sit on top of a h1 which will be on top of a div.
